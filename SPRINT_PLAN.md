# GAMBIT (갬빗) 스프린트 계획

## 개발 원칙
- **한 번에 하나의 스프린트만 진행한다.**
- 각 스프린트의 완료 기준(TEST_CRITERIA.md)을 통과해야 다음으로 넘어간다.
- 스프린트마다 **실행 가능한 결과물**이 나와야 한다(돌려볼 수 있어야 함).
- 게임 로직과 3D 화면은 **항상 분리**해서 만든다(온라인 확장 대비).
- 변경마다 즉시 커밋·푸시하고, GitHub → Vercel 자동배포로 결과를 확인한다.

전체 흐름: **두뇌(코어) → 입체 화면 → 실제 플레이 → 실사 비주얼 → 배포**

```
Sprint 1        Sprint 2        Sprint 3        Sprint 4        Sprint 5
체스 두뇌    →   입체 체스판   →   플레이 가능   →   실사 비주얼   →   배포 & 마무리
(UI 없음)       (3D 등장)       (둘 수 있음)     (와! 소리)       (실제 URL)
```

---

## Sprint 1: 체스 두뇌 (코어 엔진)
**목표:** 화면 없이도 돌아가는 "체스 규칙 + 게임 상태"를 완성한다.

**구현 항목:**
- Next.js 14 + TypeScript + Tailwind 프로젝트 셋업, GitHub 리포 생성, Vercel 연결
- 게임 상태 관리 모듈(`src/lib/chess/`) — "상태와 화면 분리"에서 **상태** 부분
- chess.js 통합: 합법 수 계산, 차례 관리
- 규칙 처리: 체크 / 체크메이트 / 스테일메이트 / 무승부(50수·3회반복)
- 특수 규칙: 캐슬링, 앙파상, 폰 승급
- 수 표현을 표준 형식(좌표 + FEN/PGN)으로 기록, 무르기(undo)
- 전역 상태 저장소(zustand 등) 골격

**산출물:** `src/lib/chess/engine.ts`, `rules.ts`, `types.ts`, `src/lib/store.ts`, 빈 화면이라도 빌드·배포되는 프로젝트 뼈대

---

## Sprint 2: 입체 체스판 (3D 렌더링 기초)
**목표:** 화면에 3D 체스판과 32개의 말이 버드아이뷰로 등장한다.

**구현 항목:**
- React Three Fiber 캔버스 구성
- 카메라: 버드아이뷰(위에서 비스듬한 탑다운), 드래그 회전·휠 줌(제한)
- 8×8 체스판 3D 메시 + 좌표계(a1~h8 ↔ 3D 위치 매핑)
- 말 8종 형태: 폰·룩·비숍·퀸·킹은 절차적 생성(lathe), 나이트는 GLTF 모델
- Sprint 1의 상태와 연결 → 현재 포지션대로 말이 제자리에 배치됨
- 기본(임시) 재질·조명으로 형태 확인

**산출물:** `src/components/board/Board3D.tsx`, `Piece.tsx`, `src/components/scene/Camera.tsx`

---

## Sprint 3: 플레이 가능 (상호작용 + 대전 모드)
**목표:** 실제로 둘 수 있다 — 한 화면 2인(핫시트) + 컴퓨터(AI).

**구현 항목:**
- 말 클릭 → 갈 수 있는 칸 하이라이트 → 클릭으로 이동
- 이동 애니메이션, 잡기 처리, 마지막 수 강조, 체크 시각 경고
- 폰 승급 선택 UI
- **로컬 2인(핫시트) 모드** — 한 기기에서 번갈아
- **vs 컴퓨터 모드** — Stockfish(WASM) 연동, 난이도(초급/중급/고급)
- 모드 선택(시작) 화면, 상태바, 컨트롤(새 게임/무르기/항복), 기보 패널, 잡은 말 표시

**산출물:** 처음부터 끝까지 한 판을 둘 수 있는 플레이 가능한 게임

---

## Sprint 4: 실사 비주얼 (폴리싱)
**목표:** "와" 소리 나는 고급 비주얼로 끌어올린다.

**구현 항목:**
- PBR 재질(원목/대리석/광택 금속), 흑·백 말 마감
- HDRI 환경 조명으로 사실적 반사, 부드러운 그림자
- 화면 후처리: 앰비언트 오클루전, 톤매핑, 약한 블룸
- 말 이동·잡기 연출 다듬기, 효과음(말 놓는 소리 등)
- 시작/결과 화면 디자인, 반응형(모바일 레이아웃)
- 성능 최적화(데스크톱 60fps / 모바일 30fps+ 목표)

**산출물:** 비주얼이 완성된 1단계 게임

---

## Sprint 5: 배포 & 마무리
**목표:** 누구나 링크로 들어와 플레이할 수 있다. 2단계(온라인) 준비를 남긴다.

**구현 항목:**
- Vercel 프로덕션 배포, (선택) 커스텀 도메인 연결
- 크로스 브라우저·기기 테스트, 버그 수정
- 로딩 최적화(3D 에셋 지연 로딩), 에러 처리
- README 작성, **2단계 온라인 대전 설계 메모**(Supabase Realtime 연결 지점 정리)

**산출물:** 배포된 게임 URL + 온라인 확장 준비 문서

---

## 향후 (별도 단계): 온라인 실시간 대전
> 1단계(Sprint 1~5)가 끝나고 진행. 로그인(카카오/구글), 방 코드·매칭, 실시간 수 동기화, 관전.
> 1단계에서 로직/화면을 분리해 두었으므로 "연결선 추가" 수준으로 붙인다.

---

## 중요 제약사항 (솔직하게)
- **완전한 실사(영화 CG급)는 불가능.** 실시간·모바일 호환을 위해 "고급 제품 사진" 수준을 목표로 한다.
- **나이트 모델**은 외부 CC0 GLTF가 필요. 적절한 모델 확보가 Sprint 2의 변수.
- **Stockfish(WASM)**는 용량이 있어 첫 로딩·메모리에 영향 → 지연 로딩 필수.
- **모바일 성능**은 기기 편차가 큼 → Sprint 4에서 품질 옵션(고/중/저) 고려.
- 3D·체스 규칙 동시 디버깅은 까다로우므로 **Sprint 1에서 로직을 화면과 분리**해 테스트 가능하게 만드는 것이 전체 성패의 핵심.
