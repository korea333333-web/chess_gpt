# Sprint 3 Playable Game Design

## Scope

Build Sprint 3: the board becomes playable. Users can choose local hotseat or computer mode, select pieces, see legal destination highlights, move pieces, undo, resign, reset, and see result/status information. Online play, real Stockfish WASM loading, polished animation, sound, and final PBR/HDRI visual polish remain out of scope.

## Architecture

Chess legality remains in `src/lib/chess`. The new `src/lib/game/controller.ts` owns UI selection decisions as pure functions so 3D click behavior is testable without React. `src/lib/ai/stockfish.ts` defines the AI boundary and provides a legal-move engine with three difficulty levels. This file is intentionally named for the future Stockfish integration point, but it does not bundle a GPL WASM asset in this sprint.

The zustand store becomes the single client-side game session state: mode, difficulty, selected square, legal target squares, last move, result/resignation state, and the current engine snapshot. React Three Fiber components receive those values as props and emit square clicks back to the store.

## Data Flow

The page renders a client `GameScreen`. `GameScreen` reads `useGameStore()`, passes the current snapshot and UI state to `Board3D`, and displays side panels. `Board3D` renders 64 clickable squares and pieces. A click calls `selectSquare(square)` in the store. The store uses controller helpers to decide whether to select, move, clear selection, or reject the click. All actual moves still go through the Sprint 1 engine.

In computer mode, after a human move leaves the game active and it is the AI side's turn, the store asks `chooseComputerMove(snapshot, difficulty)` for a legal move and applies it after a short delay.

## AI Strategy

Difficulty levels are implemented without external assets:

- Beginner: deterministic first legal move.
- Intermediate: prefer captures and promotions by simple material value.
- Advanced: prefer captures/promotions/checkmate-like high-value moves using the same material heuristic.

Every AI move comes from legal moves generated by `chess.js`, so it cannot break rules. Replacing the implementation with real Stockfish later will not affect UI components.

## UI Behavior

The first screen is now the game screen, not a landing page. Mode and difficulty controls sit in a compact side panel. The board remains the primary first-viewport element. Highlight colors show selected square, legal targets, and last move. Results show checkmate, draw, or resignation.

## Testing

Jest covers pure controller behavior and AI legal move selection. Existing chess and coordinate tests continue to run. Visual/browser verification is limited by the local in-app browser environment, so `npm run build` and HTTP checks are the primary automated verification here.
